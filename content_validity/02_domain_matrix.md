# Awadhi Domain Matrix

The complete scope of "Awadhi" the app is trying to teach. Reviewer should use this as a reference for identifying **coverage gaps** — areas of the language that the app under-covers or omits entirely.

The expectation is **NOT** that every cell is fully covered — the app is a beginner-to-intermediate learning tool. But the reviewer should be able to say "X is an essential area of Awadhi that the app barely touches" or "Y is well-covered."

## How to use this matrix during review

After rating all 195 items in the spreadsheet, scan this matrix and identify:

- **Cells where the app has good coverage** — note in the return template's "3 Strongest Areas"
- **Cells where coverage is thin or wrong** — note in "3 Weakest Areas"
- **Cells with zero coverage that should exist** — note in "3 Critical Missing Areas"

This is the most actionable feedback the app developers will receive — it tells them exactly which Awadhi domain to build content for next.
